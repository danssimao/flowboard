# 0007 - Menu

## Objetivo

Menu flutuante que mostra os tipos de nodes disponíveis para criação.
O usuário arrasta um tipo do menu e solta no Stage para criar um node.

**Customização Completa:** O consumidor controla totalmente a aparência
via render props, enquanto o Flowboard mantém a lógica de drag and drop.

> **Compound Pattern:** No modo compound, este componente é exposto como
> `Flowboard.Menu`. No modo simples, é renderizado internamente pelo
> `Flowboard` root.

## Hook useMenuDrag

### Propósito

Expõe a lógica de drag and drop para que o consumidor possa
criar menus customizados que iniciam drags.

### Interface

```typescript
interface UseMenuDragOptions {
  nodeType: string;
}

interface UseMenuDragReturn {
  handlePointerDown: (e: React.PointerEvent) => void;
  isDragging: boolean;
}

function useMenuDrag(options: UseMenuDragOptions): UseMenuDragReturn;
```

### Implementação

```typescript
import { useCallback } from 'react';
import { useFlowboardActions } from './useFlowboardActions';
import { useFlowboard } from './useFlowboard';

export function useMenuDrag({ nodeType }: UseMenuDragOptions) {
  const actions = useFlowboardActions();
  const { state } = useFlowboard();

  const isDragging =
    state.dragState?.type === 'menu-drag' &&
    state.dragState.nodeType === nodeType;

  const handlePointerDown = useCallback(
    (e: React.PointerEvent) => {
      e.preventDefault();
      e.stopPropagation();

      actions.setDragState({
        type: 'menu-drag',
        nodeType,
        mousePos: { x: e.clientX, y: e.clientY },
      });
    },
    [nodeType, actions],
  );

  return {
    handlePointerDown,
    isDragging,
  };
}
```

### Lógica

```
1. Consumidor chama useMenuDrag({ nodeType: 'task' })
2. Hook retorna { handlePointerDown, isDragging }
3. Consumidor usa handlePointerDown no onPointerDown de qualquer elemento
4. Quando handlePointerDown é chamado:
   - Dispatch SET_DRAG_STATE com type 'menu-drag'
   - nodeType é armazenado para criação do node
5. Container detecta o drag e cria o node no drop
```

## Flowboard.Menu Props

```typescript
interface FlowboardMenuProps<TItem = Record<string, unknown>> {
  // Dados customizados do consumidor
  items: TItem[];

  // Renderiza cada item do menu
  // dragProps contém handlePointerDown para iniciar o drag
  renderItem: (item: TItem, dragProps: MenuDragProps) => React.ReactNode;

  // Renderiza o container do menu (opcional)
  renderMenu?: (children: React.ReactNode) => React.ReactNode;

  // Estilo/posicionamento
  style?: React.CSSProperties;
  className?: string;
}

interface MenuDragProps {
  handlePointerDown: (e: React.PointerEvent) => void;
  isDragging: boolean;
}
```

| Campo        | Tipo                             | Obrigatório | Descrição                                   |
| ------------ | -------------------------------- | ----------- | ------------------------------------------- |
| `items`      | `TItem[]`                        | Sim         | Lista de itens do menu (dados customizados) |
| `renderItem` | `(item, dragProps) => ReactNode` | Sim         | Função que renderiza cada item              |
| `renderMenu` | `(children) => ReactNode`        | Não         | Função que renderiza o container            |
| `style`      | `CSSProperties`                  | Não         | Estilo inline no container                  |
| `className`  | `string`                         | Não         | Classe CSS no container                     |

## Comportamento

### Posicionamento

- `position: absolute` dentro do Container (fora do Stage)
- Posição: canto superior direito (`top: 12px, right: 12px`)
- z-index alto para ficar acima do Stage
- Não se move com pan/zoom (fixo relativo ao Container)

### Itens

- Para cada item em `items`, chama `renderItem(item, dragProps)`
- `dragProps.handlePointerDown` inicia o drag and drop
- `dragProps.isDragging` indica se este item está sendo arrastado
- Consumidor controla totalmente a aparência

### Drag & Drop (Pointer Events)

```
1. Usuário faz pointerDown em um item customizado
2. handlePointerDown (do useMenuDrag) dispara:
   - SET_DRAG_STATE { type: 'menu-drag', nodeType, mousePos }
3. Container detecta o drag e mostra preview via Target
4. Usuário move o mouse para dentro do Stage
5. Se pointerUp DENTRO do Stage:
   a. Converte coordenada screen→stage usando screenToStage()
   b. Aplica snap-to-grid se habilitado
   c. Gera UUID para o novo node (generateId())
   d. Dispatch ADD_NODE com:
      - id: generateId()
      - position: coordenada convertida (centro do node)
      - data: objeto vazio (consumidor preenche via onNodesChange)
   e. Dispatch SET_DRAG_STATE null
   f. Callback onNodesChange é chamado
6. Se pointerUp FORA do Stage:
   a. Dispatch SET_DRAG_STATE null (cancela)
```

### Validação

- Node só é criado se o drop ocorrer dentro do Stage (não no Menu nem no Footer)
- Posição do drop é o canto superior esquerdo do node (offset pelo tamanho default)
- Se snap-to-grid está ativo, a posição é alinhada à grade

## Implementação

### Componente Flowboard.Menu

```typescript
import { useMemo } from 'react'
import { useMenuDrag } from '../../hooks/useMenuDrag'
import type { FlowboardMenuProps } from '../../types'
import styles from './Menu.module.css'
import classNames from 'classnames'

export function FlowboardMenu<TItem extends Record<string, unknown>>({
  items,
  renderItem,
  renderMenu,
  style,
  className,
}: FlowboardMenuProps<TItem>) {
  // Renderiza o conteúdo do menu
  const content = useMemo(() => (
    <div className={classNames(styles.menu, className)} style={style}>
      <div className={styles.title}>Nodes</div>
      <div className={styles.items}>
        {items.map((item, index) => (
          <MenuItem key={index} item={item} renderItem={renderItem} />
        ))}
      </div>
    </div>
  ), [items, renderItem, style, className])

  // Aplica renderMenu se fornecido
  if (renderMenu) {
    return renderMenu(content)
  }

  return content
}

// Componente interno para cada item
function MenuItem<TItem>({
  item,
  renderItem,
}: {
  item: TItem
  renderItem: FlowboardMenuProps<TItem>['renderItem']
}) {
  // Extrai nodeType do item (assume que tem uma propriedade nodeType)
  const nodeType = (item as any).nodeType ?? String((item as any).id)
  const { handlePointerDown, isDragging } = useMenuDrag({ nodeType })

  return (
    <div className={classNames(styles.item, isDragging && styles.itemDragging)}>
      {renderItem(item, { handlePointerDown, isDragging })}
    </div>
  )
}
```

## Criação do Node (no Container/Stage)

A lógica de criação do node quando o drop acontece é tratada no Container:

```typescript
// Dentro do handlePointerUp do Container:
if (state.dragState?.type === 'menu-drag') {
  const containerRect = containerRef.current?.getBoundingClientRect();
  if (!containerRect) return;

  const mouseX = e.clientX - containerRect.left;
  const mouseY = e.clientY - containerRect.top;

  // Verificar se está dentro do stage
  if (mouseX >= 0 && mouseY >= 0) {
    // Converter para coordenadas do stage
    const stagePos = screenToStage({ x: mouseX, y: mouseY });

    // Centralizar o node na posição do drop
    const defaultSize = {
      width: theme.node.minWidth,
      height: theme.node.minHeight,
    };
    let position = {
      x: stagePos.x - defaultSize.width / 2,
      y: stagePos.y - defaultSize.height / 2,
    };

    // Aplicar snap se habilitado
    if (state.snapEnabled) {
      position = snapToGrid(position, theme.stage.gridSize);
    }

    actions.addNode({
      id: generateId(),
      position,
      data: {},
    });
  }

  actions.setDragState(null);
}
```

## Testes

### useMenuDrag

- Retorna handlePointerDown que dispara SET_DRAG_STATE
- handlePointerDown previne default e stopPropagation
- isDragging é true quando o nodeType está sendo arrastado
- isDragging é false quando não está arrastando

### Flowboard.Menu

- Renderiza todos os itens usando renderItem
- Chama renderItem com item e dragProps
- dragProps contém handlePointerDown e isDragging
- renderMenu é chamado quando fornecido
- Aplica className e style customizados
- Menu tem z-index alto (acima do stage)
- Menu não se move com pan/zoom do stage

### Integração

- Clicar em um item inicia drag (SET_DRAG_STATE dispatch com type 'menu-drag')
- Drag state contém o nodeType correto do item
- Mouse position é capturada corretamente no pointerDown
- Drop dentro do stage cria node com posição correta
- Drop fora do stage não cria node
- Node criado tem UUID único (generateId)
- Node criado tem data vazio (consumidor preenche)
- snap-to-grid é aplicado quando habilitado
